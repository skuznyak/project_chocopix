# Перевірка роботи інтеграції з Новою Поштою

## Запуск

1. Запустіть backend:
```bash
cd backend
npm run dev
```

2. Запустіть frontend:
```bash
cd frontend
npm run dev
```

## Тестування

### 1. Перевірка API Нової Пошти
```bash
# Отримати області
curl http://localhost:3000/api/np/areas

# Отримати міста
curl "http://localhost:3000/api/np/cities?areaRef=71508131-9b87-11de-822f-000c2965ae0e"

# Отримати відділення
curl "http://localhost:3000/api/np/warehouses?cityRef=8d5a92a0-34c3-11db-8d82-001a92567626"

# Розрахувати вартість доставки
curl "http://localhost:3000/api/np/delivery-cost?senderCityRef=8d5a92a0-34c3-11db-8d82-001a92567626&receiverCityRef=8d5a92a0-34c3-11db-8d82-001a92567626"
```

### 2. Перевірка frontend
1. Відкрийте http://localhost:5174
2. Додайте товари в кошик
3. Перейдіть на сторінку оформлення http://localhost:5174/checkout
4. Оберіть область, місто, відділення
5. Перевірте чи розраховується вартість доставки

## Можливі проблеми

### Сторінка оформлення порожня
1. **Порожній кошик** - додайте товари
2. **Помилка завантаження областей** - перевірте консоль браузера
3. **Не запущений backend** - переконайтесь, що backend працює на порту 3000

### Не розраховується вартість доставки
1. **Не обрано місто** - вартість розраховується після вибору міста
2. **Помилка API** - перевірте консоль браузера та логи backend

## Зміни

### Backend
- Додано ендпоінти для розрахунку вартості доставки
- API ключ Нової Пошти винесено в змінну середовища `NOVA_POSHTA_API_KEY`

### Frontend
- Додано автоматичний розрахунок вартості доставки
- Додано перевірку на порожній кошик
- Оновлено типи для збереження `cityRef`, `branchRef`, `deliveryCost`
