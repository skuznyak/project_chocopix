## Findings

### Medium: prerender fallback validation is outdated and can miss homepage SEO regressions

Files:
- [scripts/prerender-frontend.mjs](/var/www/chocopix/scripts/prerender-frontend.mjs#L7)
- [HomePage.tsx](/var/www/chocopix/frontend/src/pages/HomePage.tsx)

Problem:
- `HOMEPAGE_TITLE` in the prerender validator is still set to `Шоколадні бомбочки з маршмелоу — ChocoPix`.
- The actual homepage title is already different.
- Because of that, the validation no longer reliably catches cases where non-home pages accidentally fall back to the current homepage title.

Impact:
- SEO regressions in prerender output can slip through CI/build checks.
- This is not a live indexing bug right now, but it weakens technical SEO protection.

Recommendation:
- Sync `HOMEPAGE_TITLE` with the real homepage title.
- Better: stop hardcoding it and validate against the actual homepage render output.

### Low: orphaned `CupsPage` SEO file conflicts with actual routing

Files:
- [frontend/src/pages/CupsPage.tsx](/var/www/chocopix/frontend/src/pages/CupsPage.tsx#L1)
- [frontend/src/App.tsx](/var/www/chocopix/frontend/src/App.tsx#L26)

Problem:
- A full SEO-configured page for `/cups` still exists.
- In routing, `/cups` is not rendered; it redirects to `/cacao-bombs`.

Impact:
- Not a live indexation issue today, because the route redirects.
- But this is misleading for future maintenance and can cause accidental duplicate SEO logic if someone restores the page or adds it to sitemap/prerender later.

Recommendation:
- Either remove `CupsPage.tsx`, or explicitly mark it as deprecated in code comments and keep only the redirect path.

### Low: sitemap `lastmod` is build-date based, not content-change based

Files:
- [scripts/generate-sitemap.mjs](/var/www/chocopix/scripts/generate-sitemap.mjs#L21)
- [frontend/public/sitemap.xml](/var/www/chocopix/frontend/public/sitemap.xml#L1)

Problem:
- All URLs receive the same `lastmod`, equal to the date when the sitemap was generated.
- This does not reflect real page or product update dates.

Impact:
- Search engines can treat `lastmod` as low-trust if it changes globally without actual content changes.
- This usually does not break indexing, but it weakens sitemap quality.

Recommendation:
- Use actual content update timestamps for products/pages if available.
- If not available, it is safer to omit `lastmod` than to mass-update every URL on each generation.
